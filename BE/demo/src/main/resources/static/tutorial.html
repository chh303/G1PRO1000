Effektive algoritmer

Effektive algoritmer reduserer kjøretid og strømforbruk. En god algoritme kan være forskjellen på millisekunder og minutter.

Sammenligning av to algoritmer:

// Dårlig løsning:
for (int i = 0; i < n; i++) {
  for (int j = 0; j < n; j++) {
    if (data[i] == data[j]) {}
  }
}

// Bedre løsning med HashSet:
Set<Integer> set = new HashSet<>();
for (int val : data) {
  if (set.contains(val)) {}
  else set.add(val);
}

Quiz: Hva er fordelen med å bruke en HashSet i eksempelet over?

Den reduserer kompleksiteten fra O(n²) til O(n), og bruker mindre CPU.

Optimalisert kode

Optimalisering handler om mer enn hastighet. Det handler også om å bruke færre ressurser og redusere miljøpåvirkningen.

Eksempel på unødvendig kompleks kode:

// Ikke optimalisert:
String result = "";
for (String s : list) {
  result += s;
}

// Bedre løsning med StringBuilder:
StringBuilder sb = new StringBuilder();
for (String s : list) {
  sb.append(s);
}
String result = sb.toString();
